# Projet GenericDAO - Résumé

## Description du Projet
GenericDAO est une implémentation générique du pattern DAO (Data Access Object) en Java, conçue pour simplifier l'accès aux données dans les applications Java. Ce projet fournit une couche d'abstraction réutilisable pour les opérations CRUD (Create, Read, Update, Delete) sur différentes entités.

## Objectifs Principaux
- Fournir une interface générique pour les opérations CRUD
- Réduire la duplication de code dans les couches d'accès aux données
- Faciliter la maintenance et l'évolutivité des applications
- Standardiser l'accès aux données à travers l'application

## Structure du Projet
- src/
  - dao/
    - GenericDAO.java (Interface générique)
    - GenericDAOImpl.java (Implémentation de l'interface)
  - utils/
    - ConnectionManager.java (Gestion des connexions à la base de données)
  - entities/
    - Entités métier

## Technologies Utilisées
- Java
- JDBC pour l'accès aux données
- MySQL comme système de gestion de base de données
- Maven pour la gestion des dépendances

## Instructions d'Utilisation
1. Configurer la connexion à la base de données dans ConnectionManager
2. Créer une classe d'entité qui représente votre table
3. Étendre GenericDAOImpl pour votre entité spécifique
4. Utiliser les méthodes CRUD fournies :
   - create()
   - findById()
   - findAll()
   - update()
   - delete()

## Avantages
- Réduction du code boilerplate
- Maintenance simplifiée
- Standardisation des accès aux données
- Flexibilité et réutilisabilité

## Prérequis
- JDK 8 ou supérieur
- MySQL Server
- Maven

## Configuration
Assurez-vous de configurer correctement les paramètres de connexion à la base de données dans le fichier de configuration approprié.
